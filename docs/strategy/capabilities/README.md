# Krafted For You Capability Library

Date: `2026-05-24`

This folder contains implementation-ready output templates and progress-tracking templates for each moat capability stream.

The current data constraint is important: Krafted For You has not stored structured customer data for the past 10 years. The first version of these capabilities should therefore use a `reconstruction-first` approach:

1. start from existing hamper and workshop photos,
2. add founder memory,
3. anonymize customer details,
4. capture only enough information to learn patterns,
5. use the templates prospectively for all new inquiries.

## Capability Folders

| Capability | Folder | Primary purpose |
|---|---|---|
| Customer Emotion Intelligence | [customer_emotion_intelligence](customer_emotion_intelligence/) | Understand recipient, relationship, occasion, and desired feeling |
| Curated Option Architecture | [curated_option_architecture](curated_option_architecture/) | Turn briefs into 2-3 tasteful, feasible option paths |
| Bespoke Reliability System | [bespoke_reliability_system](bespoke_reliability_system/) | Make custom work reliable, bounded, and trackable |
| Aesthetic And Craft Standards | [aesthetic_and_craft_standards](aesthetic_and_craft_standards/) | Make outputs consistently premium and recognizably Krafted For You |
| Sourcing Intelligence | [sourcing_intelligence](sourcing_intelligence/) | Convert local market and vendor knowledge into reusable memory |
| Proof And Story Engine | [proof_and_story_engine](proof_and_story_engine/) | Turn orders and workshops into proof, stories, and content assets |
| Customer Memory And Repeat Loop | [customer_memory_and_repeat_loop](customer_memory_and_repeat_loop/) | Build repeat gifting, reminders, referrals, and continuity |

## How To Use

For each capability:

1. Open the capability folder.
2. Read that folder's `README.md`.
3. Fill the output files in the order listed there.
4. Use the `progress_and_feedback_loop.md` and `success_metrics.md` files to track whether the capability is improving.
5. Review monthly and update the templates.

## Required Context For A New Human Or Agent

Before implementing any capability, read:

1. [../moat_and_capability_strategy_2026.md](../moat_and_capability_strategy_2026.md)
2. this file,
3. the capability-specific `README.md`,
4. the specific output template being filled.

That context is intended to be enough for a new but reasoning-capable human or agent to execute the tactical implementation.
